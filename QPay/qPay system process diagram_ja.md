# qPayシステムプロセス図

## 1. QRコードを使用した支払い

### プロセスフロー

#### マーチャントワーカー
1. 請求価格を計算し、マーチャントシステム経由で請求書を作成
2. QRコード請求書作成リクエストを送信
3. 作成されたQRコードをマーチャントシステムに送信
4. QRコード請求書を印刷または顧客に送信

#### マーチャントシステム
- `qPay_genInvoice/request/` - QRコード作成リクエスト
- `qPay_genInvoice/response/` - QRコード生成レスポンス

#### 顧客
1. QRコードを受け取り、マーチャントに表示
2. QRコード請求書をスキャン
3. 支払いの意思決定
   - 「はい」の場合：支払い口座を選択し、追加情報を入力
   - 「いいえ」の場合：プロセス終了
4. 支払いボタンを押す
5. 続行ボタンを押す

#### スマートバンク
1. スマートバンクシステムにユーザーがログインしているか確認
   - ログインしていない場合：銀行顧客ログイン画面へ
2. QRコードテキストのPrefix（7729）を確認し、銀行システムに送信
3. 請求書情報を表示
4. 以下の情報を顧客に表示：
   - 追加情報入力欄
   - 支払い口座リスト
   - 受取口座情報
5. スマートバンクから受信した情報をqPayシステムに転送

#### 銀行システム
1. `qPay_decryptInfo/request/` - QRコードテキストを解析し、請求者を確認
2. 請求者とQRコードが要件を満たしているか確認
   - 満たしていない場合：要件不足の情報を銀行システムに送信
3. スマートバンクに請求書情報を提供
4. 追加情報の確認とqPayシステムへの返答
5. 支払い情報を銀行システムに転送
6. 口座残高の確認
   - 残高不足の場合：顧客に通知
7. 取引実行の可否を確認
8. 取引パスワード入力画面を表示
9. 取引を実行

#### qPayシステム
1. QRコードを解析し、要件を確認
2. `qPay_decryptInfo/response/` - 請求書情報、受取口座、追加情報を銀行システムに提供
3. `qPay_customerAction/request/` - 各種アクションリクエスト処理
   - type=1,2: 基本アクション
   - type=3,4,5,6: 支払い関連アクション
   - type=7: 取引確認
4. 追加情報の検証（マーチャントシステムからの取得が必要な場合）
5. 支払い実行可否の判定
6. 取引結果の通知

### 取引完了後
- 取引成功時：マーチャントシステムに通知
- `qPay_customerAction/response/` - 各種レスポンス処理

## 2. レシート取得（QRコード）

### プロセスフロー
1. 顧客：レシート取得アクションを実行
2. スマートバンク：レシート取得リクエストを銀行システムに転送
3. 銀行システム：レシート取得リクエストをqPayシステムに転送
4. qPayシステム：
   - `qPay_customerAction/request/type=10` - レシート取得リクエスト
   - 請求番号で確認し、QRレシートを生成
   - 銀行システムに転送
5. 銀行システム：レシートを受け取り、スマートバンクに転送
6. スマートバンク：レシートを顧客に表示

## 3. 税務レシート取得

### プロセスフロー
1. 顧客：税務レシート取得アクションを実行
2. スマートバンク：税務レシート取得リクエストを銀行システムに転送
3. 銀行システム：税務レシート取得リクエストをqPayシステムに転送
4. qPayシステム：
   - `qPay_customerAction/request/type=12` - 税務レシート取得リクエスト
   - 税務レシートが登録されているか確認
   - 登録されている場合：税務システムにリクエストを転送
   - 税務レシートを銀行システムに送信
5. 税務システム：税務レシートをqPayシステムに提供
6. `qPay_customerAction/response/type=4` - 税務レシートレスポンス
7. スマートバンク：税務レシートを顧客に表示

## 4. 支払い履歴からの削除

### プロセスフロー
1. 顧客：支払い履歴から削除アクションを実行
2. スマートバンク：削除リクエストを銀行システムに送信
3. 銀行システム：削除リクエストをqPayシステムに転送
4. qPayシステム：
   - `qPay_customerAction/request/type=8` - 削除リクエスト
   - 該当ユーザーの支払い履歴から削除
   - 銀行システムに通知
5. 削除成功の確認
   - 成功：支払いリストから削除して表示
   - 失敗：エラーメッセージ表示
6. `qPay_customerAction/response/type=4` - 削除レスポンス

## 5. qPay口座一覧

### プロセスフロー
1. 顧客：qPay口座アクションを押す
2. スマートバンク：qPay口座一覧取得リクエストを銀行システムに送信
3. 銀行システム：リクエストをqPayシステムに転送
4. qPayシステム：
   - `qPay_customerData/request/type=2` - 口座一覧リクエスト
   - ユーザー別に口座一覧を確認して送信
5. `qPay_customerData/response/type=2` - 口座一覧レスポンス
6. スマートバンク：qPay口座一覧を顧客に表示
7. 顧客がqPay口座を選択：
   - QRコード画像取得リクエスト
   - `qPay_customerAction/request/type=13` - QRコード取得
   - `qPay_customerData/response/type=3` - QRコードレスポンス
8. 取引履歴の確認（オプション）

## 6. qPay口座作成

### プロセスフロー
1. 顧客：qPay口座追加ボタンを押す
2. 作成する口座情報を入力
3. 口座作成ボタンを押す
4. スマートバンク：情報の完全性を確認
5. 銀行システム：口座作成リクエストをqPayシステムに転送
6. qPayシステム：
   - `qPay_customerAction/request/type=9` - 口座作成リクエスト
   - 情報を登録してqPay口座を作成
   - 作成したQRコードを銀行システムに送信
7. `qPay_customerAction/response/type=3` - 口座作成レスポンス
8. スマートバンク：QRコードを顧客に表示

## 7. 取引履歴

### 取引履歴表示・更新
1. 顧客：取引履歴表示ボタンを押す
2. スマートバンク：取引履歴取得リクエストを銀行システムに送信
3. 銀行システム：リクエストをqPayシステムに転送
4. qPayシステム：取引履歴情報を銀行システムに提供
5. `qPay_customerData/response/type=3` - 取引履歴レスポンス
6. スマートバンク：取引履歴を顧客に表示
7. 更新が必要な場合：
   - 顧客：取引履歴更新ボタンを押す
   - 同様のプロセスで最新情報を取得・表示
