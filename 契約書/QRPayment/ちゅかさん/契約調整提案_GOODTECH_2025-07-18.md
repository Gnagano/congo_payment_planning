# 契約書調整提案

## 1. 著作権について（第18条）

### 現状の課題
弊社は本システムをエンドユーザー様に納品する立場にあり、エンドユーザー様への納品条件と貴社との契約条件の整合性を確保する必要があります。

現在の契約条項では、以下の実務上の問題も発生します：
- 緊急のバグ修正時も貴社の許可が必要
- システム障害時の緊急対応が制限される
- 他の保守業者への引き継ぎが不可能
等

### ご提案
```
第18条について、以下のいずれかの形式をご検討いただけないでしょうか：

【案1: 共同所有】
「新規開発部分の著作権は甲乙の共同所有とし、各自が自由に利用、
改変、第三者への利用許諾（エンドユーザーへの譲渡を含む）を
行えるものとする。」

【案2: 甲への帰属】
「新規開発部分の著作権は甲（弊社）に帰属し、甲はこれを
エンドユーザーに譲渡できるものとする。
既存技術の著作権は乙が保持し、甲およびエンドユーザーに対して、
本システムの利用、改変、再配布（銀行システム内）、
第三者への保守委託を含む包括的な利用権を付与する。」

【案3: 所有権と著作権の分離】
「成果物の所有権は最終的にエンドユーザーに移転する。
著作権は乙が保持するが、甲およびエンドユーザーに対して、
以下を含む永続的かつ譲渡可能なライセンスを付与する：
・無制限の使用権
・改変権
・銀行システム内での再配布権
・第三者への保守委託権」
```

### 代替案
もし上記が難しい場合は、最低限として：
```
「甲は成果物に対して改変、保守、機能追加を行う権利を有し、
これをエンドユーザーに提供できるものとする。
エンドユーザーは銀行システムとして運用するために必要な
すべての権利（複製、改変、再配布を含む）を有するものとする」
```

## 2. 検査期間について（第15条）

### 現状
現在の契約書では「10日以内」となっていますが、営業日か暦日かが明確ではありません。
また、納品時の検査事項が決まっていない段階で10日という期間設定は少々短いと考えております。

### ご提案
```
検査期間を「20営業日以内」と明記していただけないでしょうか。
もしくは、検査項目を事前に合意し、その内容に応じた適切な期間を設定する。
```

### 理由
- 納品時の検査事項が未確定の中での期間設定はリスクが高い
- QR決済システムは銀行の基幹システムと連携するため慎重な検証が必要
- 実働日での検査期間を確保
- 検査項目の複雑さに応じた現実的な期間設定

### 配慮事項
- 明らかな不具合は発見次第即座にご連絡
- 重要な検証項目は事前に共有

## 3. 準拠法・管轄について（追加）

### 現状
現在の契約書には準拠法・管轄に関する条項が含まれていません。

### ご提案

```
・準拠法: 日本法
・管轄裁判所: 東京地方裁判所
```

### 理由
- 法的安定性の確保
- 紛争時の迅速な解決
