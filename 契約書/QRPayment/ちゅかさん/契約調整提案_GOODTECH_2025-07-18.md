# 契約書調整提案

## 1. 著作権について（第18条）

### 現状の理解
貴社が既存技術を活用されることは十分理解しており、その権利を尊重いたします。

### ご提案
```
新規開発部分については共同所有とし、
弊社は銀行システムとしての運用・改修に必要な
すべての権利を持つ形にできないでしょうか。
```

### 理由
- 銀行システムは継続的な改修・機能追加が必要
- 将来的な保守・運用の柔軟性確保
- 貴社の既存技術は引き続き保護されます

### 代替案
もし共同所有が難しい場合は、最低限以下の権利をご許可いただけないでしょうか：
- 改変権（バグ修正、機能追加）
- 再配布権（銀行システム内での展開）
- 第三者への保守委託権

## 2. 検査期間について（第15条）

### 現状
現在の契約書では「10日以内」となっていますが、営業日か暦日かが明確ではありません。

### ご提案
```
検査期間を「10営業日以内」と明記していただけないでしょうか。
```

### 理由
- QR決済システムは銀行の基幹システムと連携するため慎重な検証が必要
- 実働日での検査期間を確保
- 次期プロジェクトのスケジュールにも配慮した現実的な期間

### 配慮事項
- 明らかな不具合は発見次第即座にご連絡
- 重要な検証項目は事前に共有
- 効率的な検証プロセスで双方の負担を軽減
