# QR決済機能拡張予定
*2025年7月2日時点*

## 拡張機能概要

初回実装完了後に追加予定の機能を、下記の実装順序に従って段階的に展開します。


## 拡張機能一覧表

| 機能名         | 予算                     | 開発期間                        | 実装スケジュール例※1    | 実装順序 | 備考                                             |
| ----------- | ---------------------- | --------------------------- | -------------- | ---- | ---------------------------------------------- |
| **加盟店機能**   | 2,948万円                | 計2.5ヶ月<br>（β版1.5ヶ月+リリース1ヶ月） | 2026年1月〜3月中旬   | 1    | Static QR以外の全トランザクション種別対応含む※2                  |
| **カード決済機能** | 1,650万円                | 2ヶ月                         | 2026年3月中旬〜5月中旬 | 2    | 連携カードシステムに依存<br>カードシステム改修が必要な場合あり              |
| **海外決済機能**  | 550～1,100万円<br>/決済システム | 連携先次第                       | 2026年6月以降順次    | 3    | Alipay、WeChat、UnionPay、PayPay等<br>コンゴサポート状況に依存 |
|             |                        |                             |                |      |                                                |

**【補足1】** 初回実装が2025年8月開始で12月完了を想定した場合

**【補足2】トランザクション種別の詳細**
- **Invoice**: 請求書ベースの決済(ポストペイド)
- **Product**: 商品別QR決済
- **In-app**: 銀行アプリを呼び出して行う決済
- **Dynamic QR**: 動的QRコード生成による決済(駐車場などで固定のQRを読み込んだ際に車番情報などで金額が動的に変更されるQR)
- **POS QR**: POS端末連携QR決済(POS端末で生成するワンタイムQR)

## 実装順序の理由

### 1. 加盟店機能を最初に実装
- カード決済は主に加盟店向け機能となるため、加盟店管理基盤が必要
- 加盟店機能実装により、後続のカード決済導入が容易になる

### 2. カード決済機能を次に実装
- 海外決済の多くはカード決済機能をベースとしている
- カード決済基盤により、海外決済システムとの連携が効率化

### 3. 海外決済機能を最後に実装
- 前段階の機能を活用して効率的に実装可能
- 連携先のサポート状況に応じて柔軟に対応

## 個人向けサービス（都度検討）

初回実装から除外された以下の機能は、現地事業者との連携協議に応じて都度検討：

| カテゴリ | サービス内容 |
|---------|------------|
| **公共サービス** | 税金支払い、電子レシート発行・管理、公共料金支払い（電気・ガス・水道・通信） |
| **生活サービス** | モバイルトップアップ、交通機関決済、IPTVサービス料金、サブスクリプション管理 |

※ 実装順序に特別な考慮は不要、パートナー協議状況に応じて柔軟に対応
